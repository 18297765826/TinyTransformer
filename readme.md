#  1.Transformer 为什么要除以 sqrt(d_k)？不除会怎样？
## 点积结果方差随着dim线性增大，除以 sqrt(d_k)可以将方差归一化为1，并且softmax对大数值敏感，这么做可以防止softmax进入饱和区导致梯度消失
#   2.MultiHead Attention 相比单头有什么好处？
## 多头将特征空间拆成多个子空间并行建模，每个头学习不同的依赖关系，如一个头关注句法依存，另一个头关注语义相似性，另一个头关注位置距离，表达能力比单头强，并且计算量不变
#   3.头数拆的越多越好吗？
## 头数和每头维度之间存在权衡，头数过多导致每个子空间维度太低，反而损失表达能力，需要根据 dim 大小选择合适的头数。
#   4.LayerNorm 和 BatchNorm 的区别是什么？Transformer 为什么用 LayerNorm？
##  LayerNorm 对单样本所有特征维度归一化，不依赖 batch 内其他样本，天然适合序列长度可变、batch size 小的 NLP 场景。
#   5.Pre-Norm 和 Post-Norm 哪个更好？各有什么缺点？
##  Post-Norm在残差相加之后再归一化。训练初期梯度不稳定，深层网络容易梯度消失，通常需要 warmup 才能稳定训练。Pre-Norm先归一化再过子层。表达能力理论上略弱于 Post-Norm，最终性能上限稍低。Pre-Norm 的残差路径是干净的，梯度可以直接通过残差连接回传到底层，训练更稳定，不太需要 warmup。Post-Norm 的归一化包住了残差，梯度路径更复杂，但每层输出都被约束得更好，模型容量更强。
#   6.位置编码为什么用 sin/cos？用可学习的位置编码会怎样？
##  三点原因：第一是可以表达相对位置，第二是正余弦函数值域在[0,1]之间，不会随着位置增加而爆炸，第三点是正余弦函数是连续函数，对比训练序列更长的位置也能直接计算，不像可学习位置编码只能覆盖预设的max_len
#   7.Decoder 为什么需要两个 MHA？causal mask 的作用是什么？
##  第一个 MHA：Decoder 对目标序列做自注意力，但加了 causal mask，确保位置 t 只能看到 0~t-1 的 token，模拟自回归生成的过程。如果不加 mask，训练时每个位置都能看到答案，推理时却只能逐步生成，训练和推理行为不一致，模型就废了。第二个 MHA：q 来自 Decoder，k 和 v 来自 Encoder 输出。本质是让 Decoder 的每个位置去查询 Encoder 的全部信息，决定当前生成的 token 应该关注源序列的哪些部分。这也是 seq2seq 中对齐的核心机制。
#   8.Transformer 的复杂度是多少？序列长度对计算量的影响？
##  slef-attention复杂度是O(n²d),FFN复杂度是O(nd²)，n是序列长度，d是维度,叠加L层后总复杂度是O(L(n²d+nd²)),因此，短序列FFN占主导，长序列Attention占主导
#   10.你的实现里 Encoder 和 Decoder 共用了同一个 token_emb，这样合理吗？
##  同语言场景下共用合理，语义一致且节省参数；跨语言场景下词表不同，必须分开。
#   11.训练时 Decoder 用 teacher forcing，推理时怎么做？两者有什么差异？
##  两者的问题：Exposure Bias，训练时永远看到正确的上文，推理时却要依赖自己的输出。一旦某一步生成错了，错误会随着序列累积传播，训练和推理的分布出现偏差，这个问题叫做 Exposure Bias。Scheduled Sampling：训练时以一定概率用模型自己的输出替代真实标签，让模型适应自己的错误，强化学习：直接用生成结果的最终得分训练，绕过 teacher forcing
#   12.如果要支持更长的序列，位置编码怎么改？
##  RoPE：LLaMA、GPT-NeoX 用的方案，把位置信息编码成旋转矩阵作用在 q 和 k 上，天然编码相对位置，外推能力比 sin/cos 强很多。现在是大模型的主流选择。ALiBi不直接编码位置，而是在注意力分数上加一个和距离成比例的惩罚项，距离越远惩罚越大，外推性能非常好。
